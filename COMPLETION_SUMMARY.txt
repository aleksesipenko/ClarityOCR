ROOT CAUSE ANALYSIS
================================================================================

THREE CRITICAL BUGS IDENTIFIED:

1. STAGE CONTEXT MANAGER BUG (CRITICAL)
   Location: pipeline_v2.py lines 53-70, _stage_context()
   Problem: Stage never transitions to next phase
   - Sets progress_pct=100 within current stage
   - Never advances to next stage (ocr→polish, polish→done)
   - File forever stuck at stage="ocr"
   Impact: API shows job incomplete even though it IS processing
   
2. INDENTATION BUG (CRITICAL)
   Location: pipeline_v2.py lines 986-989
   Problem: Success transition code unreachable
   - Success code at wrong indentation level
   - If any exception occurs, code never executes
   - File status never transitions to "completed"
   - Job appears running forever
   Impact: Jobs finish but API never shows completion
   
3. SINGLE-THREADED WORKER BLOCK (MAJOR)
   Location: pipeline_v2.py lines 216-340, PipelineWorker class
   Problem: One worker thread blocks on I/O
   - Polish subprocess blocks entire worker
   - Multiple jobs serialize, cannot progress in parallel
   - Queue backs up, appears like system-wide hang
   Impact: Multiple jobs all appear stuck at 99%

KEY INSIGHTS
================================================================================

1. Bug #1 (Stage Stuck): Graceful degradation can hide failures
   - Polish silently fails (degraded mode)
   - But stage never transitions
   - Looks like "hanging" when really just stuck in one phase

2. Bug #2 (Unreachable Code): Python indentation is critical
   - Success code at wrong scope level
   - Unreachable when exceptions occur
   - Classic Python pitfall

3. Bug #3 (I/O Block): ThreadPoolExecutor elegantly solves
   - Frees main worker thread from I/O blocking
   - Enables concurrent processing
   - Maintains GPU constraint (subprocess is serialized)

4. Investigation Lesson: The 99% was a red herring
   - Actually 100% progress within current stage
   - Real issue was stage never advancing
   - Led investigation down wrong path initially
